# 访问统计说明（给作者自己看）

这套统计是**第一方、隐私友好**的：复用你自己的 Cloudflare KV，只存几个累加的整数计数，
**不碰用户的咖啡数据、不记 IP、不写 Cookie、无法定位到任何人**。比多数第三方统计都干净。

---

## 一、怎么看数据（最简单）

部署后，直接在浏览器打开：

```
https://coffee-log-b2o.pages.dev/api/track
```

会看到一段 JSON，长这样：

```json
{
  "visit": 312,
  "first_record": 86,
  "aha_5cups": 24,
  "share_card": 11,
  "sync_on": 9,
  "since": "2026-06-07"
}
```

`since` 是开始统计的日期。其余每个数字 = **有多少台不同设备走到了这一步**
（每台设备每个事件只算一次，所以约等于"独立用户数"）。

---

## 二、这几个数字对应推广计划的漏斗

| 字段 | 含义 | 漏斗环节 | 看它能判断 |
|------|------|----------|-----------|
| `visit` | 打开过应用的独立设备数 | 获取 | 推广带来了多少人 |
| `first_record` | 录入了第一杯记录的人 | **激活** | 多少人真的上手了（光看不录=没激活） |
| `aha_5cups` | 记满 5 杯、看到分析的人 | **北极星(NSM)** | 多少人到达"产品真正有用"的时刻 |
| `share_card` | 生成过分享卡片的人 | 自传播 | 增长引擎转得快不快 |
| `sync_on` | 开了云同步的人 | 重度用户 | 有多少"真爱粉"（想跨设备用） |

**怎么用**：算相邻两环的转化率，哪一环掉得最狠就去补那一环。比如：
- `first_record / visit` 低 → 落地页/首屏没说清价值，来了就走。
- `aha_5cups / first_record` 低 → 用户录了一两杯就放弃，门槛太高（可加"示例数据/快速上手"）。
- `share_card` 很低 → 阿哈时刻的"晒卡片"引导不够显眼。

---

## 三、隐私边界（也写进了 README）

- 只上报**事件名**（如 `aha_5cups`），不带任何内容、不带咖啡数据。
- 后端（`functions/api/track.js`）只把对应计数 +1，**不存 IP、不存时间戳、不写 Cookie**。
- 每台设备每个事件**只报一次**（用 localStorage 标记去重），所以数据库写入量极低，
  稳稳在 Cloudflare 免费额度内。
- 打点完全**不阻塞、不报错**：网络或接口出问题就静默跳过，绝不影响 App 使用。

## 四、（可选）再加一层"渠道来源"统计——强烈建议

上面的第一方统计能告诉你"多少人走到哪一环"，但**说不清他们从哪个平台来的**。
而推广计划最关键的一个问题就是：**到底小红书、V2EX、少数派哪个真的带人？**

**Cloudflare Web Analytics** 正好补上这块：免费、无限量、隐私优先（不写 Cookie、符合 GDPR），
能看 **访问量 + 来源referrer + 国家/地区**。它和你现在的 Cloudflare 部署天然搭配。

开启只需三步（一次性，约 5 分钟）：

1. 登录 Cloudflare 控制台 → 左侧 **Analytics & Logs → Web Analytics** → **Add a site**，
   填你的域名 `coffee-log-b2o.pages.dev`。
2. 它会给你一段 `<script ... data-cf-beacon='{"token":"你的token"}'></script>`。
3. 把那段脚本粘到 `index.html` 的 `</body>` 之前（和现有 `<script>` 放一起），
   重新部署即可。之后在控制台就能看到流量和来源。

> 提示：如果是 Cloudflare Pages 自带域名，部分账户能在 Pages 项目设置里一键开启 Web Analytics，
> 不用手动贴脚本——可以先去 Pages 项目的 **Settings → Analytics** 看看有没有开关。

## 五、改动后记得重新部署

这套统计是新加的后端函数 + 前端打点，**必须重新部署到 Cloudflare Pages 才会生效**
（本地直接开 `index.html` 时 `/api/track` 不存在，打点会静默失败，不影响使用）。
部署方式见 [部署云同步.md](部署云同步.md)，和之前一样 `git push` 或 `wrangler pages deploy` 即可。
