# 手冲手记 · 升级设计：味道调整建议 + dial-in 调试轨迹

日期：2026-06-05

## 背景

软件目前是「记录本」（记录过去的冲煮），缺少精品咖啡玩家真正想要的
「调试助手」能力——用味道反推参数、一杯杯把同一包豆调到最好喝、并能复现。
本次新增两个功能填补这一点。

## 功能1：味道 → 下次怎么调

### 交互
- 在「记录一杯」弹窗里加一组复选框（**可多选**）：发酸 / 发苦 / 寡淡 / 太浓 / 涩。
- 勾选后，弹窗内**实时**显示「💡 下次试试」方向建议。
- 保存后，该条冲煮卡片上显示勾选的味道（chips）+「💡 下次试试」建议块；
  多选则多行，每个问题一行。

### 味道 → 建议对照表（给方向，不给具体数字）

| key | 味道问题 | 建议 |
|-----|---------|------|
| sour | 发酸、尖锐、像没熟（萃取不足） | 研磨调细一点 · 水温升 2~3°C · 延长冲煮时间 |
| bitter | 发苦、焦、喉咙发干（萃取过度） | 研磨调粗一点 · 水温降 2~3°C · 缩短冲煮时间 |
| weak | 寡淡、像水、没味道（太稀） | 粉水比调浓（如 1:15→1:14）或加点粉 |
| strong | 太浓、压舌、化不开（太浓） | 粉水比调淡（如 1:15→1:16）或加点水 |
| astringent | 涩、收敛、舌头发紧 | 研磨调粗一点 · 水温略降 · 注水轻柔少搅拌 |

## 功能2：单豆详情页 dial-in

### 🎯 最佳配方卡片
- 取该豆**评分最高**的一杯（同分取**最近**的）。
- 展示：粉水比 / 水温 / 研磨 / 闷蒸 / 总时间 / 器具。
- 按钮「照这个再冲一杯」：弹出记录窗，参数预填，日期=今天，
  评分清零、笔记清空、味道问题清空。
- 该豆**没有任何打分**时不显示此卡片。

### 调试轨迹
- 该豆冲煮按**时间正序**排成竖线，每个点显示：评分 + 器具 + 研磨度 + 水温。

### 保留
- 原「按评分高→低」的冲煮列表保留在轨迹下方。

## 数据模型

- 冲煮记录新增字段 `tasteIssues: string[]`（如 `["sour","astringent"]`）。
- 向下兼容：老记录无该字段，读取/渲染时用 `brew.tasteIssues || []` 兜底，无需迁移。

## 影响文件
- `index.html`：记录弹窗加味道复选框 + 建议实时显示区。
- `app.js`：TASTE_ISSUES 配置、建议渲染、最佳配方计算、复现、调试轨迹渲染。
- `style.css`：复选框组、建议块、最佳配方卡片、调试轨迹的样式。

## 不做（YAGNI）
- 不做具体磨豆机刻度联动（只给方向）。
- 不做味道提示在「复现」时的二次提醒。
- 不做自动「比上一杯改了什么」对比。
