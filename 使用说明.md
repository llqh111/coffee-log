# ☕ 手冲手记 · 使用说明

一个帮你记录和优化手冲咖啡的本地网页应用。数据默认存在浏览器里，也可以设个同步码在多台设备间共享。

## 怎么打开

### 在线使用（推荐）
👉 **https://llqh111.github.io/coffee-log/**

手机浏览器打开即可。想装到手机主屏幕像 App 一样用：
- **安卓 Chrome**：菜单 → 「添加到主屏幕 / 安装应用」
- **iPhone Safari**：分享按钮 → 「添加到主屏幕」

### 本地使用
直接**双击 `index.html`**，浏览器打开即用。不需要安装，也不需要联网。

## 怎么用

### 1. 先加一包豆（「咖啡豆」标签）
点 **+ 新增豆子**，填名称、烘焙日期、产地、处理法、烘焙度等。
- 填了**烘焙日期**，软件会自动算「**养豆天数**」（今天 − 烘焙日），每天自动更新。
- 填了购买克数，会在豆子卡片显示余量进度条。

### 2. 记录每一杯（「冲煮记录」标签）
点 **+ 记录一杯**：
- 选这杯用的豆子 → 填粉量、水量 → **粉水比自动算**（如 1:15.0）
- 还能记：水温、研磨度、闷蒸（水量 + 时间）、总冲煮时间、器具
- **打分（1–10）** 并写**风味笔记**——这是以后找规律的关键
- 勾选这杯的缺点（如发酸、发苦），下方会**实时给出调整方向**

### 3. 单豆详情（点豆子卡片进入）
- 看这包豆的冲煮次数、平均分、最高分
- **🎯 最佳配方**：自动挑出这包豆打分最高的一杯，列出全部参数，可一键「照配方再冲一杯」
- **🧪 调试轨迹**：按时间线展示每次的参数变化，直观看到你在怎么调

### 4. 找规律（「分析」标签）
切到「分析」标签（要先给几杯打过分）：
- **🏆 高分配方**：自动挑出评分最高的那些杯，总结参数集中在哪
- **💡 下次试试**：基于高分杯，给出具体调参方向
- **⚖️ 分类对比**：按器具 / 处理法分组比平均分，看哪类你更擅长

打分的杯数少于 3 杯时，会提示「仅供参考」——多记几杯结论才准。

## 云同步（多设备共享）

点击右上角的同步指示器，设一个**同步码**（随便一串字，别太简单），所有设了相同同步码的设备会自动合并数据。

- 不同设备上打开同一个网址，设相同的同步码即可
- 删除的记录会同步（墓碑机制），不会复活
- 同步码留空并保存 = 断开同步，数据保留在本地
- 同步码像密码一样，**不要告诉别人**

## 备份与换设备

- 点右上角 **导出备份** → 得到一个 `coffee-log-日期.json` 文件
- 换电脑或清浏览器缓存后，点 **导入** 选这个文件恢复

> ⚠️ 如未启用云同步，数据只存在当前浏览器里。**换浏览器、清缓存、用无痕模式**都会读不到数据，请养成偶尔导出备份的习惯。

## 文件说明

| 文件 | 作用 |
|---|---|
| `index.html` | 页面结构 |
| `style.css` | 样式（颜色、排版） |
| `app.js` | 核心逻辑（记录、计算、存取） |
| `merge.js` | 多设备数据合并规则 |
| `sync.js` | 云同步客户端 |
| `sw.js` | 离线缓存（PWA） |
| `manifest.webmanifest` | PWA 配置（添加到主屏幕用） |
| `icon.svg` | 图标 |

所有文件放在**同一个文件夹**里即可。想调颜色，打开 `style.css` 最上面的 `:root` 区域改色值。
